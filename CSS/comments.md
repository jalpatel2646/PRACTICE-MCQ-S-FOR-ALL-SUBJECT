## 3. CSS Comments

**Q21. Correct way to write a CSS comment?**


A. // comment
B. <!-- comment -->
C. /* comment */
D. # comment #

**Answer: C**
##

**Q22. CSS comments can be:**


A. Only single-line
B. Only multi-line
C. Both single-line and multi-line
D. Only inside <style> tag
**Answer: B**

**Q23. Comments in CSS are:**
A. Executed by browser
B. Ignored by browser
C. Shown as tooltip
D. Converted to HTML


**Answer: B**
##


**Q24. CSS comments can be used in:**


A. Inline CSS only
B. Internal CSS only
C. External CSS only
D. All of the above


**Answer: D**
##

**Q25. Purpose of CSS comments:**


A. Make code readable
B. Temporarily disable code
C. Documentation
D. All of the above
**Answer: D**

**Q26. Can comments be nested in CSS?**
A. Yes
B. No
C. Only in external CSS
D. Only in inline CSS


**Answer: B**
##

**Q27. Comment syntax in CSS starts with:**


A. //
B. /*
C. <!--
D. #


**Answer: B**
##

**Q28. Comment syntax in CSS ends with:**


A. //
B. */
C. -->
D. #


**Answer: B**
##

**Q29. CSS comments are ignored during:**


A. Parsing
B. Execution
C. Rendering
D. All of the above


**Answer: D**
##

**Q30. CSS comments can contain:**


A. Only text
B. Text and code
C. Only code
D. Nothing

**Answer: B**
